01

Payrexx — Ticket-Checkout

💳
Payrexx Whitelabel
Bestehende go2flow Integration — Checkout läuft auf Payrexx, Webhook informiert das Backend
1
User wählt Ticket-Kategorie
Auf d2c-summit.ch/tickets — Kategorien mit Preisen und verfügbaren Plätzen. Optional: Voucher-Code eingeben (Preis wird live angepasst).
2
Persönliche Daten eingeben
Vorname, Nachname, Firma, Funktion — erscheinen später auf Ticket und Badge.
3
CheckoutService erstellt Payrexx Gateway
Laravel ruft Payrexx API auf: CheckoutService::initiate() → erstellt Gateway mit Betrag, Beschreibung und Redirect-URL.
POST Payrexx API → Gateway-URL zurück
4
User zahlt auf Payrexx Whitelabel-Checkout
User wird zu Payrexx weitergeleitet — Zahlung per Kreditkarte, TWINT, etc. Payrexx übernimmt die Zahlungsabwicklung vollständig.
5
Payrexx sendet Webhook bei erfolgreicher Zahlung
PayrexxWebhook::handle() empfängt die Bestätigung:
  • Order-Status → paid
  • Ticket generieren (mit eindeutigem QR-Code)
  • Ticket-PDF erstellen (asynchron via Job)
  • Bestätigungs-E-Mail mit PDF-Anhang senden
  • Kontakt in Marketing-Engine aufnehmen
6
User landet auf Bestätigungsseite
Redirect zurück zu d2c-summit.ch/tickets/bestaetigung — mit Social Share Card Angebot ("Share on LinkedIn").
Voucher-Direktlink
Für Direkteinladungen (VIP, Partner, Gratis-Tickets) kann ein Direktlink generiert werden:
d2c-summit.ch/tickets?voucher=SUMMIT-VIP-2026
→ Nuxt lädt die Seite, validiert den Voucher via API, passt den Preis an und füllt den Code vor. Normaler Checkout-Flow ab hier.
02

Mailtrap — Transaktionsmails

📬
Sending API
Alle System-E-Mails über Mailtrap Sending API
  • Ticket-Bestätigung — mit Ticket-PDF als Anhang
  • Session-Buchungsbestätigung
  • Wartelisten-Benachrichtigung (Platz frei)
  • Passwort-Reset
  • E-Mail-Verifikation bei Registrierung
Laravel Mail Config
MAIL_MAILER=mailtrap
MAILTRAP_API_TOKEN=xxx
MAILTRAP_INBOX_ID=xxx
# INBOX_ID nur für Testing
Mailtrap-Account ist bereits vorhanden (go2flow). API-Token muss eingetragen werden.
03

Mailtrap — Marketing & Kampagnen

📣
Email Marketing API
Kampagnen werden über Mailtrap Marketing API abgewickelt — Kontakte synchronisiert, Kampagnen versendet, Stats zurückgeholt
Was das Backend macht
  • Kontakte an Mailtrap synchronisieren (mailtrap_contact_id speichern)
  • Kampagne via API versenden (mailtrap_campaign_id speichern)
  • Stats periodisch zurückholen (Laravel Scheduler)
  • Unsubscribes aus Mailtrap-Webhook übernehmen
Mailtrap API Calls
POST /api/contacts → sync
POST /api/send → versenden
GET /api/campaigns → stats

Automatisierte E-Mail-Sequenzen

Trigger E-Mail Timing Typ
ticket_purchased Bestätigung + Ticket-PDF als Anhang Sofort Transaktional
session_booked Masterclass-Buchungsbestätigung Sofort Transaktional
waitlist_promoted Platz frei — Einladung zur Buchung Sofort Transaktional
pre_event_7d "Dein Tagesplan" + praktische Infos 7 Tage vor Event Automation
pre_event_1d Reminder + Programm-Highlights 1 Tag vor Event Automation
post_event_1d Danke + "Präsentationen jetzt verfügbar" Tag nach Event Automation
feedback_request Bitte um Session-Bewertung 3 Tage nach Event Automation
Alle Sequenzen sind im Admin konfigurierbar — aktivierbar/deaktivierbar per Event. Die Trigger werden durch Laravel Events/Listeners ausgelöst und als Jobs in die Queue gestellt.
05

QR Check-in Flow

📱
Check-in am Eventtag
Kein App-Download — Browser-QR-Scanner, mobiloptimiert
1
Check-in-Personal öffnet checkin.d2c-summit.ch
Login mit Rolle checkin — sieht nur die Scanner-Oberfläche, nichts sonst
2
QR-Code des Attendees scannen
Browser-Kamera wird aktiviert — QR-Code vom Ticket (PDF oder My Summit) scannen
3
Laravel validiert das Ticket
POST /api/v1/checkin/scan { qr_code } → gibt Ticket-Infos zurück: Name, Firma, Kategorie, gebuchte Masterclasses
4
Check-in bestätigen
Ticket-Status wird auf checked_in gesetzt, checked_in_at und checked_in_by_user_id werden gespeichert
✓
Fehlerzustände abgefangen
Ungültiger QR-Code / Ticket bereits eingecheckt / Ticket storniert → klare Fehlermeldung. Bei Offline: lokaler Cache als Fallback.
