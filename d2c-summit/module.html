Core Platform — Event, Programm & Speaker
9 Stories
🔁
M01 v1.0
Event Engine
Kernmodul — jedes andere Modul ist einem Event zugeordnet.
US-01-1
Als Organizer möchte ich ein neues Event anlegen (Titel, Datum, Venue, Status: Entwurf/Publiziert/Archiviert), damit alle weiteren Inhalte einem Event zugeordnet werden können.
MVPP1M
US-01-2
Als Organizer möchte ich Stages und Räume definieren (Name, Kapazität, Reihenfolge), damit Sessions konkret verortet werden können.
MVPP1S
US-01-3
Als Organizer möchte ich Event-Branding konfigurieren (Logo, Primärfarbe, Hero-Bild), damit die öffentliche Website das Erscheinungsbild des Events widerspiegelt.
MVPP2S
US-01-4
Als Organizer möchte ich ein vergangenes Event klonen, damit ich für das Folgejahr nicht alles neu aufbauen muss.

MVPP2M
US-02-2

MVPP2S
US-02-3
Als Besucher möchte ich das Tagesprogramm nach Stage und Typ filtern, damit ich schnell die für mich relevanten Sessions finde.
MVPP2S
US-02-4

Unternehmens-Profile

Als Organizer möchte ich Unternehmen anlegen (Name, Logo, Website, Typ: Sponsor / Attendee-Firma), damit Teilnehmer und Sponsoren einer Firma zugeordnet werden können.
MVPP3M
US-04-2
Als Organizer möchte ich die Website-Sichtbarkeit pro Unternehmen steuern (Sponsor-Seite / Partner / unsichtbar), damit ich kontrolliere was auf der öffentlichen Website erscheint.
MVPP3S
US-04-3
Als Organizer möchte ich mehrere Tickets einem Unternehmen zuordnen, damit Firmen-Gruppen (z.B. Brand bucht 2 Tickets) sauber abgebildet werden.
MVPP3S
US-04-4
Als Organizer möchte ich im Admin nach Unternehmenstyp, Sichtbarkeit und Paket filtern, damit ich schnell den richtigen Datensatz finde.
MVPP3XS

MVPP3S
US-05-2

MVPP3S
US-05-3
Als Organizer möchte ich Teilnehmer nach Unternehmen, Ticket-Kategorie und Check-in-Status filtern, damit ich gezielt suchen und auswerten kann.
MVPP3S
US-05-4

Rollen & Rechte
Vollständige Berechtigungs-Architektur von Tag 1 — 7 Rollen, pro Event vergeben.
US-07-1
Als Super Admin möchte ich Benutzer anlegen und ihnen Rollen pro Event vergeben (super_admin / organizer / sponsor_manager / attendee / speaker / sponsor / checkin), damit jeder nur auf seinen Bereich zugreifen kann.
MVPP1M
US-07-2
Als System müssen alle 4 Domains (public, my, admin, checkin) durch Middleware rollenbasiert geschützt sein, damit kein unbefugter Zugriff möglich ist.
MVPP1L
