1
Woche 1–2
Foundation
Technisches Fundament — alles was nachfolgende Phasen brauchen
Setup Infrastruktur
Backend
  • Laravel-Projekt aufsetzen (LTS)
  • Auth & Sanctum konfigurieren
  • Rollen-System (7 Rollen, per Event)
  • Vollständiges MySQL-Schema (alle Migrations)
  • Domain-Struktur & API-Routing (v1)
Frontend
  • Nuxt 4 Projekt aufsetzen
  • Tailwind CSS + Design-System
  • Pinia + API-Layer-Grundstruktur
  • Auth-Store (Login, Logout, Token)
  • Base-Layouts: Public, My Summit, Admin
DevOps
  • GitHub Repo + Branch-Strategie
  • CI/CD Pipeline (GitHub Actions)
  • Staging-Umgebung aufsetzen
  • Environment-Variablen strukturieren
🏁
Meilenstein: Projekt-Setup komplett — Dev-Team kann parallel arbeiten
2
Woche 3–4
Event Engine + Programm + Public Website
Das Herzstück — Events, Programm und erste öffentliche Seite
Module 01–03 Modul 11
Admin Backend
  • Event CRUD (erstellen, bearbeiten, archivieren, klonen)
  • Venue + Stages verwalten
  • Branding-Config pro Event
  • Sessions CRUD + Drag & Drop Reihenfolge
  • Speaker CRUD (Admin-seitig)
  • Speaker ↔ Session verknüpfen
Public Website (SSR)
  • Home-Seite (Hero, Event-Info, Highlights)
  • Programm-Seite (Tagesplan, Filter)
  • Speaker-Übersicht + Detailseite
  • SEO: Schema.org Event, OG-Tags
  • FAQ-Seite (via Admin pflegbar)
Speaker-Portal
  • Speaker-Login (über bestehendes Auth)
  • Eigenes Profil bearbeiten
  • Präsentationen hochladen
🌐
Meilenstein: Public Website ist live — Event, Programm und Speaker sichtbar
3
Woche 5–6
Ticket-System & Zahlungsintegration
Tickets kaufen, Payrexx-Integration, Voucher-System, erste E-Mails
Modul 04 Payrexx Mailtrap
Ticketing
  • Ticket-Kategorien CRUD (Admin)
  • Checkout-Flow: Kategorie → Daten → Payrexx → Bestätigung
  • QR-Code-Generierung (unique, fälschungssicher)
  • Ticket-PDF generieren (Browsershot)
  • Order-Management im Admin
  • Stornierung + Rückerstattungs-Workflow (Admin)
Voucher-System
  • Voucher CRUD (Admin)
  • Validierung via API
  • Direktlink-Funktion (?voucher=CODE)
  • Preis-Anpassung im Checkout
  • Voucher-Statistiken im Admin
E-Mail (Mailtrap)
  • Mailtrap-Anbindung (transaktional)
  • Ticket-Bestätigung (mit PDF-Anhang)
  • Rechnung als PDF
  • Passwort-Reset-Mail
💳
Meilenstein: Erster echter Ticketkauf möglich — Payrexx → Bestätigung → PDF per Mail
4
Woche 7–8
My Summit — Mitgliederbereich
Persönlicher Bereich für Attendees — Ticket, Programm, Badge, Social Cards
Modul 05 Modul 12 Modul 13 Abnahme I
My Summit Core
  • Registrierung + Login (Auth-Flow)
  • Dashboard: Ticket, Tagesplan, Materialien
  • Ticket-Ansicht mit QR-Code + PDF-Download
  • Rechnungsdownload
  • Profil bearbeiten
Session-Buchung
  • Vollständiges Programm im My Summit
  • Masterclass buchen (mit Platz-Anzeige)
  • Session stornieren
  • Warteliste bei voller Session
  • Automatische Benachrichtigung bei freiem Platz
Generierte Inhalte
  • Badge-Generator (PDF A6 + PNG)
  • Social Share Card (LinkedIn + Instagram)
  • Share-Button beim Ticketkauf
  • Materialien-Bereich (Präsentationen)
⭐
Abnahme I — Ticket-Kauf bis My Summit ist vollständig: Käufer kann sein Ticket sehen, buchen, Badge generieren
5
Woche 9–10
Sponsor-Portal
Self-Service für Sponsoren + öffentliche Sponsoren-Seite
Modul 07
Admin-seitig
  • Sponsor-Pakete definieren (Platin–Partner)
  • Leistungen pro Paket (JSON konfigurierbar)
  • Sponsor einer Firma zuweisen + Paket zuordnen
  • Sponsor-Materialien hinterlegen (Toolkit, Guidelines)
Sponsor Self-Service
  • Sponsor-Login (über bestehendes Auth)
  • Logo hochladen (PNG + SVG)
  • Firmenbeschreibung, Website-URL, Kontakt pflegen
  • Materialien downloaden
  • Statistiken: Logo-Impressions, Link-Klicks
Public Website
  • Sponsoren-Seite (Logo + Link + Beschreibung)
  • Automatische Rang-Sortierung (Platin oben)
  • Logo-Impression-Tracking
6
Woche 11–12
Marketing Engine
Kontakt-Management, Kampagnen, Automationen, UTM-Tracking
Modul 08 Mailtrap
Kontakt-Management
  • CSV-Import (Kontakte aus Event 2025)
  • Auto-Import aus Ticket-Käufen
  • Tags + Segmentierung
  • Kontaktlisten / Segments erstellen
  • DSGVO: Unsubscribe-Link, Consent-Flag
Kampagnen
  • E-Mail Template-Editor (HTML + Variablen)
  • Kampagne erstellen + Ziel-Liste zuweisen
  • Mailtrap Marketing API anbinden
  • Test-Mail senden, Vorschau
  • Versand sofort oder geplant (scheduled)
  • Stats: Sent / Opened / Clicked / Bounced
Automationen & UTM
  • Automatisierte Sequenzen (7 Trigger)
  • Pre-Event 7d + 1d Reminder
  • Post-Event: Danke + Materialien
  • UTM-Link Generator pro Kampagne
  • Shortlinks + Klick-Tracking
  • Konversions-Tracking: Kampagne → Ticketkauf
📧
Meilenstein: Erste Marketing-Kampagne kann versendet werden — Import + Segmentierung + Versand
7
Woche 13–14
Statistics, Check-in & Finalisierung
Alle verbleibenden Module — Statistiken, Check-in App, Zertifikat, Feedback
Modul 09 Modul 10 Modul 14 Abnahme II
Statistics Dashboard
  • Admin Dashboard mit Live-Daten
  • Ticket-Verkaufs-Charts (nach Kategorie)
  • Session-Auslastung
  • Marketing-Stats (Kampagnen-Rückholung von Mailtrap)
  • Check-in Rate + Vorjahresvergleich
Check-in Web-App
  • QR-Scanner via Browser-Kamera
  • Ticket validieren + bestätigen
  • Fehlerzustände (ungültig, bereits eingecheckt)
  • Offline-Fallback (lokaler Cache)
  • Live Check-in Stats für Einlass-Team
Finalisierung
  • Certificate of Attendance (PDF-Generierung)
  • Zertifikat in My Summit verfügbar
  • Session-Feedback System (Post-Event Trigger)
  • Feedback aggregiert in Statistics
  • Badge-Vorlagen im Admin konfigurierbar
✅
Abnahme II — Feature Complete: alle 14 Module fertig, Platform vollständig nutzbar

Woche 15–16
