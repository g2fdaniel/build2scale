Phasenplan im Detail

1
Woche 1–2
Foundation
Technisches Fundament + vollständige Rollen-Architektur von Tag 1
Setup M07 Rollen
Backend
  • Backend-Projekt aufsetzen
  • Authentication & Autorisierung konfigurieren
  • Rollen-System: 7 Rollen, pro Event vergeben
  • Vollständiges Datenbankschema (alle MVP-Module)
  • Domain-Struktur & API-Routing (4 Domains)
Frontend
  • Frontend-Projekt aufsetzen
  • Design-System & Komponenten-Basis
  • State Management + API-Anbindung
  • Auth-State (Login, Logout, Session)
  • Base-Layouts: Public / My Summit / Admin
DevOps
  • Code-Repository + Branch-Strategie
  • CI/CD Pipeline einrichten
  • Staging-Umgebung aufsetzen
  • Hosting & Deployment konfigurieren
🏁
Setup komplett — Team kann parallel arbeiten, Rollen-Architektur steht
2
Woche 3–4
Event Engine + Programm + Public Website
Herzstück der Plattform — Events, Programm und erste öffentliche Seiten
M01 Event M02 Programm M03 Speaker M12 Website
Admin Backend
  • Event CRUD (erstellen, bearbeiten, archivieren)
  • Venue + Stages / Räume verwalten
  • Branding-Config pro Event (Logo, Farbe)
  • Sessions CRUD (Typ, Stage, Zeit, Kapazität)
  • Speaker CRUD (Admin-seitig — kein Speaker-Login)
  • Speaker ↔ Session verknüpfen
  • FAQ-Verwaltung im Admin
Public Website
  • Home-Seite (Hero, Event-Info, Highlights)
  • /programm — Tagesplan mit Filter (Stage/Typ)
  • /speaker — Grid + Detailseiten
  • /faq — aus Admin gespeist
  • SEO: Schema.org Event-Markup, OG-Tags
🌐
Public Website mit Event, Programm und Speakern ist live
3
Woche 5–6
Unternehmens-Profile + Teilnehmer + Ticket-System
Neues Kernmodul: Unternehmen als Entität — plus Ticketkauf via Payrexx
M04 Unternehmen ★ Neu M05 Teilnehmer M06 Tickets
Unternehmens-Profile (Neu)
  • Unternehmen anlegen: Name, Logo, Website, Typ
  • Sichtbarkeit auf Website steuern
  • Tickets einem Unternehmen zuordnen
  • Admin-Übersicht: Filter nach Typ, Paket, Sichtbarkeit
Ticket-System & Payrexx
  • Ticket-Kategorien CRUD (Admin)
  • Payrexx-Integration: Checkout → Bestätigung
  • QR-Code-Generierung (unique, fälschungssicher)
  • Ticket-PDF + Bestätigungsmail (Mailtrap)
  • Order-Management: Status, Rechnungsdownload
  • Stornierung + Rückerstattungs-Workflow
  • Public Website: /tickets Seite live
Teilnehmer-Verwaltung
  • Teilnehmer-Übersicht im Admin
  • Unternehmens-Zuordnung (Teilnehmer ↔ Unternehmen)
  • Filter + Export (CSV)
💳
Erster echter Ticketkauf möglich — Payrexx → QR → PDF → Bestätigungsmail
4
Woche 7–8
Sponsor-Verwaltung + Kommunikations-Basis + Website komplett
Website vollständig, erste Kommunikation mit Teilnehmern möglich
M09 Sponsoren M10 Kommunikation Abnahme I
Sponsor-Verwaltung
  • Sponsor-Pakete definieren (Platin/Gold/Silver/Bronze)
  • Unternehmen als Sponsor markieren + Paket zuweisen
  • Logo-Upload direkt im Admin
  • /sponsoren Seite: Logos nach Paket-Rang
  • Voucher-System (% + Gratis + Direktlink)
Kommunikations-Basis
  • E-Mail Templates im Admin pflegbar
  • Gezielte Updates an Teilnehmer-Gruppen senden
  • Filter: alle / nach Ticket-Kategorie / nach Unternehmen
  • Versand-Übersicht: wer hat was erhalten
⭐
Abnahme I — Website komplett live · Ticketkauf funktioniert · Sponsoren sichtbar · Kommunikation möglich
5
Woche 9–10
My Summit Basis + Check-in App + Statistiken
Teilnehmer-Portal (Fundament) und Einlass-Tool für den Eventtag
M08 My Summit M13 Check-in M11 Stats
My Summit — Basis
  • Login / Registrierung (verknüpft mit Ticketkauf)
  • Ticket-Ansicht: QR-Code, Daten, PDF-Download
  • Profil bearbeiten (Name, Firma, Funktion)
  • Programm-Übersicht (Lese-Ansicht)
Check-in App
  • QR-Code Scanner (Browser-Kamera)
  • Ticket validieren: Name, Firma, Kategorie
  • Fehlermeldungen (ungültig / bereits eingecheckt)
  • Live-Einlass-Zähler
Admin — Basis-Statistiken
  • Ticket-Verkäufe gesamt + nach Kategorie
  • Umsatz + Voucher-Einlösungen
  • Check-in Rate (live am Eventtag)
6
Woche 11–12
QA, Security & Go-Live
End-to-End Testing aller MVP-Flows — dann Produktion
Testing Security Abnahme II
End-to-End Testing
  • Ticket-Flow: Kauf → QR → Check-in
  • Admin-Flow: Event anlegen → Website live
  • Kommunikations-Flow: Mail versenden → Anzeige
  • My Summit: Login → Ticket anzeigen
  • Cross-Browser & Mobile-Tests
Security & Performance
  • Auth-Schutz auf allen geschützten Routen
  • Payrexx-Webhook-Validierung prüfen
  • Rollen-Zugriff: alle 7 Rollen durchgetestet
  • Performance-Optimierung (Ladezeiten, Images)
  • DSGVO-Check (Daten, Mails, Consent)
Go-Live
  • DNS-Konfiguration (4 Domains)
  • Staging → Production Deployment
  • SSL, Monitoring, Backups
  • Team-Einführung + Admin-Schulung
🚀
Abnahme II — MVP live. Erster D2C Summit kann auf der Platform stattfinden.
II

Post-MVP — Phase II nach Go-Live

Diese Module sind nicht blockierend für den ersten Summit. Sie werden nach dem Go-Live in der Reihenfolge priorisiert die den grössten Wert bringt — typischerweise My Summit voll ausgebaut als erstes, da Masterclass-Buchungen dann möglich werden müssen.
