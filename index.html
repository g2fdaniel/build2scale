5 MVP-Features

🔒

Gated Entry

Alle Seiten sind ohne Login nicht sichtbar. Middleware redirected nicht eingeloggte Besucher zur Club-Login-Seite.

✉️

Invitation-Only Registrierung

Keine öffentliche Registrierung. Sicherer Token-Link per E-Mail, gültig 7 Tage, einmalig verwendbar.

🏷️

Club-Preislogik

20–50% Rabatt auf Originalpreis. UVP durchgestrichen, Club-Preis prominent — in Liste, Detail, Cart & Checkout.

👥

Mitgliederverwaltung

Mitglieder Firmen/Gruppen zuweisen. Gruppenbasierte Rabattstufen. Admin-Interface zum Verwalten.

✨

Club Splash & Login

"Exklusiv. Nur für Mitglieder." — Login-Formular mit E-Mail + Passwort und Passwort-Vergessen-Flow.

  • DB-Migration schreiben (3 neue Tabellen + Fixtures)
  • ClubMembershipService Interface definieren

W2
Backend: Auth, Invitation, Members Woche 2
  • ClubAuthMiddleware + IceHawkDelegate Integration
  • ShowClubLoginRequestHandler + DoLoginRequestHandler + DoLogoutRequestHandler
  • InvitationRepository + MemberRepository + CompanyRepository
  • InvitationService: Token-Generierung, E-Mail-Versand
  • AcceptInvitationRequestHandler: Token → Account + Member → Auto-Login
  • Unit Tests für Invitation-Flow und Auth-Middleware
W3
Frontend: Club-Oberfläche & Preislogik Woche 3
  • ClubPriceModifier implementieren
  • Twig-Templates: login.twig, invite-registration.twig
  • Club-Preisanzeige in Produktliste, Detail, Cart (Twig-Partials)
  • React-Admin: Member-Liste + Einladungsformular
  • Club-Splash-Seite: Branding, "Nur für Mitglieder"-Claim
W4
Checkout, Tests & Übergabe Woche 4
  • Club-Preise im Checkout durchziehen
  • E2E-Tests: Invitation → Registrierung → Login → Warenkorb → Checkout
  • Bugfixes + Edge Cases (abgelaufener Token, gesperrtes Mitglied)
  • Dokumentation: DB-Schema, Routen, Admin-Anleitung
  • Staging-Übergabe an Fortuneglobe
